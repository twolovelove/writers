## 기능 영역별 디자인 관리 포인트

| 기능 영역 | 디자인 관리 포인트 |
| --- | --- |
| **글쓰기 에디터**<br>(`Editor.tsx`, `EditorToolbar`) | 원고지 느낌의 몰입감, 진행률 표시(`ProgressBar`)의 심리적 동기부여 |
| **첨삭 노트**<br>(`feedback.ts`, `FeedbackPanel`) | 칭찬/제안/총평 구조가 읽기 편한지, 정보 위계 |
| **아카이브 / 모음집**<br>(`ArchiveCalendar`, `Compilation`) | 달력·목차·인쇄본 레이아웃이 실제로 "책처럼" 보이는지 |
| **로그인 & 글 데이터**<br>(`useSession`, `useEntries`, `useDraft`, `lib/entries.ts`) | 로그인 유도 문구, 저장 상태를 유저가 안심할 수 있게 노출 |
| **리뷰 위젯 & 관리자 페이지**<br>(`ReviewWidget`, `AdminReviews`) | 위젯이 방해되지 않으면서 눈에 띄는 위치/톤 |
| **설정 / 개인정보 / 탈퇴**<br>(`Settings.tsx`, `PrivacyPolicy.tsx`) | 탈퇴/삭제 같은 되돌릴 수 없는 액션의 경고 UX |

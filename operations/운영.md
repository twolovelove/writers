## 1. 기능 영역별 운영 관리 포인트

| 기능 영역 | 운영 관리 포인트 |
| --- | --- |
| **글쓰기 에디터**<br>(`Editor.tsx`, `EditorToolbar`) | 하루 1,000자 잠금 규칙(`useDailyLock`)이 기획 의도대로 동작하는지 주기 점검 |
| **첨삭 노트**<br>(`feedback.ts`, `FeedbackPanel`) | 첨삭 문구가 유저에게 상처 주지 않는 톤인지 주기적 재검토 |
| **아카이브 / 모음집**<br>(`ArchiveCalendar`, `Compilation`) | 이 기능이 사실상 "데이터 내보내기" 대체 수단임을 인지([로드맵.md](./로드맵.md) 참고) |
| **로그인 & 글 데이터**<br>(`useSession`, `useEntries`, `useDraft`, `lib/entries.ts`) | 저장 실패는 Sentry(`captureError`)로 잡힘 — DSN 설정 후 실제 발생 빈도 주기 확인. 로컬 캐시가 없어져 오프라인 자동저장은 더 이상 지원 안 됨 |
| **리뷰 위젯 & 관리자 페이지**<br>(`ReviewWidget`, `AdminReviews`) | 들어온 리뷰를 실제로 정기적으로 확인·반영하는 루틴 |
| **설정 / 개인정보 / 탈퇴**<br>(`Settings.tsx`, `PrivacyPolicy.tsx`) | 개인정보처리방침 문구가 실제 데이터 처리 방식과 항상 일치하는지(기능 바뀔 때마다 동반 수정) |
